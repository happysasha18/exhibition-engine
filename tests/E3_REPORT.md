## Obstacles

None that required stopping. Two small observations:

- `build.py` calls `shutil.rmtree(OUT)` + `OUT.mkdir()` TWICE inside `build()` (lines 437–439 and 458–459). The second call discards the first, leaving a clean dir. No data loss — nothing is written between the two calls. Not a bug but slightly confusing. Left unchanged (engine/build.py is not modified).
- The `finalist_series.json` member format `"NNNN_<id>.jpg"` is an artifact of the instance's catalog numbering (the sort prefix). This format is baked into `id_of = lambda m: m.split("_",1)[1].rsplit(".",1)[0]` in build.py. The fixture satisfies it; any real content must do the same.
